# Polair — отслеживание заявок на ремонт оборудования

Next.js-приложение: публичная страница поиска заявок по номеру заявки или
серийному номеру оборудования, плюс скрытая панель администратора для
загрузки Excel-отчётов и просмотра статистики.

## Запуск локально

```bash
npm install
npm run dev
```

Откройте http://localhost:3000

## Как это устроено

- **Главная страница (`/`)** — поле поиска. Ввод номера заявки или серийного
  номера показывает карточку с информацией о заявке.
- **Скрытый вход в админку** — если ввести в поле поиска `2218`, произойдёт
  переход на `/admin`. Отдельного логина/пароля нет (как и просили в ТЗ);
  при необходимости добавьте защиту (см. раздел «Как добавить пароль» ниже).
- **Панель администратора (`/admin`)**:
  - загрузка отчёта за 2026 год (.xlsx/.xls);
  - загрузка архива 2024–2025 (.xlsx/.xls);
  - удаление загруженных файлов;
  - вкладка «Обзор и исполнители» — выпадающий список исполнителей и период
    (7 дней / 30 дней / 2026 / 2025 / 2024), а также график заявок по дням
    сбоку с выбором периода;
  - вкладка «Поиск по столбцам» — поиск по Номенклатуре, Местонахождению,
    Заводскому №, Исполнителю, датам принятия/выполнения и Статусу;
  - вкладка «Повторные заявки» — группы заявок по одному и тому же
    оборудованию/адресу; клик по строке раскрывает историю всех обращений.

## Формат загружаемого Excel-файла

Файл должен содержать одну строку заголовков с названиями колонок, как в
образце из ТЗ (Год, Период, № заявки, Номенклатура, Заводской №,
Организация, Дата принятия заявки на ремонт, Дата выполнения ремонта,
Владелец оборудования, Местонахождение оборудования, ФИО, Телефон,
Исполнитель, Неисправность, Код неисправности по классификатору, Работы,
запчасти, транс. услуги, Отчеты, Комментарий, Итого руб, Статус заявки в
Фениксе, Кол-во, Сумма). Парсер ищет строку заголовков автоматически в
первых 10 строках листа, поэтому шапка отчёта (название/период) сверху не
мешает.

## ⚠️ Важно про хранение данных на Vercel

Загруженные файлы сохраняются в JSON-хранилище на сервере. На обычном
сервере (или при локальном запуске) это работает постоянно. **На Vercel
файловая система — только для чтения, кроме папки `/tmp`**, которая
очищается при холодном старте / новом деплое. Это значит, что после
загрузки отчёта на Vercel он будет доступен, пока функция "тёплая", но
может пропасть после длительного простоя или нового деплоя.

Для продакшн-использования на Vercel рекомендуется одно из:
1. Подключить **Vercel Blob** или **Vercel KV** для хранения загруженных
   данных (постоянное хранилище).
2. Подключить внешнюю БД (Postgres/Supabase/PlanetScale) через Vercel
   Storage.
3. Хранить строго последний загруженный файл и коммитить его в репозиторий
   вручную (не рекомендуется для частых обновлений).

Если нужно — могу подключить Vercel Blob/KV в следующей итерации, для
этого потребуется включить соответствующий Storage в проекте Vercel.

## Деплой на Vercel

1. Залейте содержимое этого репозитория на GitHub.
2. На vercel.com → New Project → импортируйте репозиторий.
3. Framework Preset — Next.js (определится автоматически).
4. Deploy.

## Как добавить пароль на /admin (опционально)

Сейчас переход на админку скрыт за "секретным" номером заявки (2218), без
дополнительной аутентификации. Если нужна защита паролем — добавьте
Middleware с Basic Auth или простую проверку пароля в `pages/admin.js`
(могу реализовать по запросу).
