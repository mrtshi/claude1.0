# Polair — отслеживание заявок на ремонт оборудования

Next.js-приложение: публичная страница поиска заявок по номеру заявки или
серийному номеру оборудования, плюс скрытая панель администратора для
загрузки Excel-отчётов и просмотра статистики.

## Запуск локально

```bash
npm install
npm run dev
```

Откройте http://localhost:3000

## Как это устроено

- **Главная страница (`/`)** — поле поиска. Ввод номера заявки или серийного
  номера показывает карточку с информацией о заявке.
- **Скрытый вход в админку** — если ввести в поле поиска `2218`, произойдёт
  переход на `/admin`. Отдельного логина/пароля нет (как и просили в ТЗ);
  при необходимости добавьте защиту (см. раздел «Как добавить пароль» ниже).
- **Панель администратора (`/admin`)**:
  - загрузка отчёта за 2026 год (.xlsx/.xls);
  - загрузка архива 2024–2025 (.xlsx/.xls);
  - удаление загруженных файлов;
  - вкладка «Обзор и исполнители» — выпадающий список исполнителей и период
    (7 дней / 30 дней / 2026 / 2025 / 2024), а также график заявок по дням
    сбоку с выбором периода;
  - вкладка «Поиск по столбцам» — поиск по Номенклатуре, Местонахождению,
    Заводскому №, Исполнителю, датам принятия/выполнения и Статусу;
  - вкладка «Повторные заявки» — группы заявок по одному и тому же
    оборудованию/адресу; клик по строке раскрывает историю всех обращений.

## Формат загружаемого Excel-файла

Файл должен содержать одну строку заголовков с названиями колонок, как в
образце из ТЗ (Год, Период, № заявки, Номенклатура, Заводской №,
Организация, Дата принятия заявки на ремонт, Дата выполнения ремонта,
Владелец оборудования, Местонахождение оборудования, ФИО, Телефон,
Исполнитель, Неисправность, Код неисправности по классификатору, Работы,
запчасти, транс. услуги, Отчеты, Комментарий, Итого руб, Статус заявки в
Фениксе, Кол-во, Сумма). Парсер ищет строку заголовков автоматически в
первых 10 строках листа, поэтому шапка отчёта (название/период) сверху не
мешает.

## Хранение данных (важно!)

Загруженные отчёты сохраняются в **Vercel Blob** (приватное постоянное
хранилище) — это решает проблему, при которой отчёты "пропадали" после
обновления страницы. Раньше данные лежали во временной папке `/tmp`,
которую Vercel может очистить в любой момент (после простоя, при
масштабировании, при новом деплое), из-за чего один и тот же сайт мог то
показывать статистику, то нет, в зависимости от того, какой именно
серверный инстанс отвечал на запрос. Теперь при наличии Vercel Blob
данные хранятся надёжно, независимо от того, какой инстанс их читает.

### Как подключить Vercel Blob (нужно сделать один раз)

1. Откройте ваш проект на vercel.com.
2. В боковом меню выберите **Storage** → **Create Database** → **Blob**.
3. Задайте имя (например, `polair-reports`), выберите **Private** доступ,
   нажмите **Create**.
4. Vercel предложит подключить хранилище к проекту (сейчас это обычно
   происходит через OIDC — Vercel сам управляет доступом без
   долгоживущего токена в переменных окружения; это нормально и более
   безопасно, чем старый способ с `BLOB_READ_WRITE_TOKEN`).
5. Сделайте новый деплой (**Redeploy**) проекта — это обязательно, иначе
   подключение может не заработать в уже запущенных серверных функциях.
6. Зайдите в `/admin` и загрузите отчёты заново — теперь они сохранятся
   постоянно.

Приложение определяет доступность Blob не по переменным окружения (они
не всегда видны напрямую при OIDC-подключении), а по **фактической
попытке** обращения к хранилищу при каждом запросе. Если попытка
проходит успешно — данные сохраняются в Blob. Если Vercel Blob не
подключён или недоступен — в панели администратора появится жёлтое
предупреждение, а данные будут храниться во временной памяти (как
раньше) — то есть проблема пропадающих отчётов актуальна до тех пор,
пока хранилище не подключено и не пройден Redeploy.

### Локальная разработка

При запуске через `npm run dev` без подключённого Blob попытка
обращения к нему завершится неудачей, и приложение автоматически
использует локальный файл на диске — Vercel Blob подключать для
разработки не обязательно.

### Альтернативы

Вместо Vercel Blob можно использовать любую БД (Postgres/Supabase и т.п.)
— для этого потребуется переписать `lib/store.js` под нужный клиент; при
необходимости могу это сделать.

## Деплой на Vercel

1. Залейте содержимое этого репозитория на GitHub.
2. На vercel.com → New Project → импортируйте репозиторий.
3. Framework Preset — Next.js (определится автоматически).
4. Deploy.
5. **Обязательно** подключите Vercel Blob, как описано выше в разделе
   «Хранение данных», иначе отчёты будут периодически пропадать.

## Как добавить пароль на /admin (опционально)

Сейчас переход на админку скрыт за "секретным" номером заявки (2218), без
дополнительной аутентификации. Если нужна защита паролем — добавьте
Middleware с Basic Auth или простую проверку пароля в `pages/admin.js`
(могу реализовать по запросу).
